# LAN API 与 VPN：使用前必须选择一种模式

> **发布版结论（已验证）**
>
> 当前版本不能把“开启 VPN 测试国外模型”和“手机 LAN API 转发给电脑”混在同一次使用中。
>
> - 要测试 OpenAI、Codex 等国外模型的账号额度：在实际发起测试的设备上开启可用的 VPN/代理。
> - 要让 Android 手机把 LAN API 转发给电脑：**手机和电脑都必须关闭全部 VPN、代理客户端和系统代理**，然后再启动服务和客户端。
>
> 任一端仍有 VPN/代理时，电脑可能看到 `HTTP 502`、连接超时或获取模型失败。手机上的“测试连接”通过，不代表电脑可以访问手机服务。

---

## 先选使用场景

| 目标 | 手机 VPN/代理 | 电脑 VPN/代理 | 可否通过 LAN API 转发给电脑 |
| --- | --- | --- | --- |
| 验证账号额度、测试国外模型 | 在**实际发请求的设备**上开启所需 VPN/代理 | 同左 | 不要同时使用 LAN API 转发 |
| 手机 LAN API 供电脑使用 | **全部关闭** | **全部关闭** | 可以，前提是手机与电脑可互访 |

如果上游模型只有开启 VPN 才能访问，那么在当前网络与客户端组合下，不能同时满足“电脑通过手机 LAN API 使用该上游”和“任一端开启 VPN”。请先使用账号验证功能完成额度测试，或等待使用不影响局域网连接的网络方案后再尝试转发。

## LAN API 转发的正确步骤

1. 完全退出手机和电脑上的 VPN/代理软件，并确认 Windows 系统代理已关闭。
2. 手机和电脑连接同一路由器的普通网络，不能使用访客网络或开启客户端隔离的 Wi-Fi。
3. 在手机的 **局域网 API** 页面开启服务，复制页面显示的 Base URL 与配对密钥。
4. 在电脑客户端填写：

   | 字段 | 填写内容 |
   | --- | --- |
   | Base URL | 完整复制手机页面的 `http://手机局域网IP:8787/v1` |
   | Key / API Key | 只填 `sk-phone-...`，不要包含 `Bearer ` |
   | 协议 | 按手机当前上游选择；Codex++ 使用 `Chat Completions` 时，手机上游也必须支持该协议 |

5. 在电脑客户端点击“从上游获取”或“诊断供应商”。成功后再选择返回的真实模型 ID。

> 手机页面上的地址才是无线 LAN 地址。不要填写 VPN 的 `tun0` IP，也不要填写 WebView 调试端口。手机服务重启后，必须重新复制新的配对密钥。

## USB 仅改变地址，不改变 VPN 规则

USB 调试可将电脑地址改为本机回环地址：

```powershell
adb devices -l
adb forward tcp:8787 tcp:8787
```

然后电脑 Base URL 填：

```text
http://127.0.0.1:8787/v1
```

这能避开 Wi-Fi 入站连接，但在当前已验证的环境中，**仍应关闭手机和电脑的全部 VPN/代理**。USB 断开、重启手机或重启 ADB 后，需要重新执行转发命令。

## `HTTP 502` 或获取模型失败

按以下顺序处理：

1. 退出电脑 VPN/代理，并关闭 Windows 系统代理。
2. 退出手机 VPN/代理。
3. 停止后重新开启手机 LAN API，重新复制 Base URL 与新生成的配对密钥。
4. 确认 Base URL 末尾为 `/v1`。
5. 在电脑客户端重新点击“从上游获取”。

仍失败时，先在 Windows PowerShell 检查端口。将地址替换成手机页面显示的值：

```powershell
Test-NetConnection <手机局域网IP> -Port 8787
curl.exe -H "Authorization: Bearer sk-phone-替换为实际密钥" `
  http://<手机局域网IP>:8787/v1/models
```

| 现象 | 处理 |
| --- | --- |
| `Test-NetConnection` 失败或超时 | 检查两台设备是否同网段、路由器是否开启 AP/客户端隔离，以及两端 VPN/代理是否已完全关闭 |
| HTTP 401 | 配对密钥已过期或填写了 `Bearer ` 前缀；重新从手机页面复制裸 `sk-phone-...` 值 |
| HTTP 502 | 首先确认两端 VPN/代理与系统代理均已关闭，再重启 LAN API 后重试 |
| `/v1/models` 成功但聊天失败 | 模型 ID 或 Responses / Chat Completions 协议与手机当前上游不匹配 |

## 为什么手机自测成功，电脑仍会失败

手机的“测试连接”验证的是：

```text
手机 App -> 当前上游模型
```

电脑使用 LAN API 时，还增加了：

```text
电脑客户端 -> 手机 LAN API -> 当前上游模型
```

VPN/代理可能会干预前一段局域网请求，即使手机到上游的测试仍然成功。因此，转发场景必须把两端 VPN/代理全部关闭后再判断服务是否正常。
